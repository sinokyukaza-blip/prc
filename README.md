# 📘 Практика з ООП (Java)

## 📌 Опис проєкту
У процесі розробки проєкту було створено програму для обчислення математичної функції, збереження та обробки отриманих результатів.  
Проєкт демонструє використання ключових принципів ООП, роботу з серіалізацією, колекціями, багатопоточністю.

## 🧮 Індивідуальне завдання
Обчислити середнє арифметичне значення функції `1000 * sin(α)` для чотирьох довільних аргументів та визначити **кількість одиниць у двійковому поданні** цілої частини отриманого результату.

## 🏗️ Структура проєкту
Проєкт містить:
- класи для обчислень;
- класи для збереження параметрів і результатів;
- механізм серіалізації та десеріалізації;
- ієрархію класів для відображення результатів;
- засоби тестування;
- механізми паралельної обробки даних.

## ⚙️ Реалізований функціонал

### ✅ Завдання 1. Серіалізація
У межах першого завдання:
- розроблено клас, що серіалізується, для зберігання параметрів і результатів обчислень;
- використано агрегування для побудови класу, що виконує розв’язання задачі;
- реалізовано збереження та відновлення стану об’єкта за допомогою серіалізації;
- продемонстровано особливості використання `transient` полів;
- створено клас для тестування правильності обчислень та серіалізації;

### 🏭 Завдання 2. Factory Method
У другому завданні:
- результати обчислень розміщено у колекції з можливістю збереження та відновлення;
- реалізовано шаблон проєктування **Factory Method**;
- створено ієрархію класів, що дозволяє розширювати способи відображення результатів;
- розроблено інтерфейс для "фабрикованих" об’єктів;
- реалізовано текстове подання результатів;
- створено інтерфейс для фабричного методу.

### 📊 Завдання 3. Табличне подання результатів
У третьому завданні:
- розширено ієрархію класами для подання результатів у вигляді текстової таблиці;
- передбачено налаштування параметрів таблиці користувачем;
- реалізовано діалоговий інтерфейс із користувачем;
- створено клас для тестування основної функціональності;

### 🔄 Завдання 4. Команди та undo
У четвертому завданні:
- реалізовано можливість скасування виконаних операцій (**undo**);
- продемонстровано використання **макрокоманд**;
- застосовано шаблон **Singleton**;
- реалізовано діалоговий інтерфейс;
- створено клас для тестування функціональності програми.

### ⚡ Завдання 5. Паралельна обробка
У п’ятому завданні:
- реалізовано паралельну обробку елементів колекції;
- забезпечено виконання операцій пошуку мінімуму, максимуму, середнього значення, відбору за критерієм та статистичної обробки;
- управління чергою завдань реалізовано за допомогою шаблону **Worker Thread**.

## 🧠 Використані принципи ООП
У проєкті застосовано такі принципи об’єктно-орієнтованого програмування:
- інкапсуляція;
- наслідування;
- поліморфізм;
- абстракція.

## 💾 Робота з даними
Програма підтримує:
- відновлення попереднього стану об’єктів;
- роботу з колекціями результатів;
- серіалізацію та десеріалізацію даних.

## 🖥️ Інтерфейс користувача
Проєкт містить консольний діалоговий інтерфейс, який дозволяє:
- вводити вхідні дані;
- запускати обчислення;
- переглядати результати в текстовому вигляді;
- скасовувати окремі операції;
- виконувати серії команд.

## 🧪 Тестування
Для перевірки роботи програми розроблено окремі класи тестування, що перевіряють:
- правильність обчислень;
- коректність серіалізації та десеріалізації;
- роботу шаблонів проєктування;
- основну функціональність програми.

## 📌 Висновок
У ході виконання практичних завдань було закріплено знання з об’єктно-орієнтованого програмування, серіалізації, роботи з колекціями, шаблонів проєктування та багатопотокової обробки.  
У результаті створено програмний проєкт, який демонструє практичне застосування теоретичних знань з Java та ООП.
